# Тестирование функциональности объявлений с помощью Postman

## Подготовка Postman

### 1. Создание коллекции
- Создайте новую коллекцию в Postman с названием "TechMarket API"
- Создайте Environment (например, "Local Development")
- Добавьте переменную `baseUrl` со значением `http://localhost:8080`

### 2. Переменные окружения
```
baseUrl: http://localhost:8080
authToken: (будет заполняться автоматически)
listingId: (будет заполняться автоматически)
```

## Этап 1: Регистрация и авторизация

### 1.1 Регистрация пользователя
- **Method**: POST
- **URL**: `{{baseUrl}}/api/auth/register`
- **Headers**:
  ```
  Content-Type: application/json
  ```
- **Body** (raw JSON):
```json
{
    "email": "test@example.com",
    "password": "password123",
    "role": "USER"
}
```
- **Ожидаемый ответ**: 200 OK с JWT токеном

### 1.2 Авторизация (получение токена)
- **Method**: POST
- **URL**: `{{baseUrl}}/api/auth/login`
- **Headers**:
  ```
  Content-Type: application/json
  ```
- **Body** (raw JSON):
```json
{
    "email": "test@example.com",
    "password": "password123"
}
```
- **Tests** (для автоматического сохранения токена):
```javascript
const response = pm.response.json();
pm.environment.set("authToken", response.token);
```

## Этап 2: Тестирование функциональности объявлений

### 2.1 Создание объявления (ВАЖНО!)
- **Method**: POST
- **URL**: `{{baseUrl}}/api/listings/add`
- **Headers**:
  ```
  Content-Type: application/json
  Authorization: Bearer {{authToken}}
  ```
- **Body** (raw JSON):
```json
{
    "title": "iPhone 14 Pro",
    "brand": "Apple",
    "price": 89990.00,
    "condition": "New"
}
```

**Формат создания объявления:**
- `title` (string): Название объявления (обязательно)
- `brand` (string): Бренд техники (обязательно)
- `price` (decimal): Цена в формате числа с плавающей точкой (обязательно)
- `condition` (enum): Состояние техники, возможные значения:
  - `"New"` - Новое
  - `"Used"` - Б/у

**Ожидаемый ответ**:
```json
{
    "id": 1,
    "title": "iPhone 14 Pro",
    "brand": "Apple",
    "price": 89990.00,
    "condition": "New",
    "status": "Active",
    "ownerId": 1,
    "createdAt": "2024-03-09T15:30:00",
    "updatedAt": "2024-03-09T15:30:00"
}
```

**Tests** (для автоматического сохранения ID):
```javascript
const response = pm.response.json();
pm.environment.set("listingId", response.id);
```

### 2.2 Получение всех объявлений
- **Method**: GET
- **URL**: `{{baseUrl}}/api/listings?page=0&size=10&sortBy=createdAt&direction=desc`
- **Headers**: Без авторизации
- **Параметры**:
  - `page` - номер страницы (начинается с 0)
  - `size` - количество элементов на странице
  - `sortBy` - поле для сортировки (createdAt, price, title)
  - `direction` - направление сортировки (asc, desc)

### 2.3 Получение конкретного объявления
- **Method**: GET
- **URL**: `{{baseUrl}}/api/listings/{{listingId}}`
- **Headers**: Без авторизации

### 2.4 Фильтрация по бренду
- **Method**: GET
- **URL**: `{{baseUrl}}/api/listings/brand/Apple`
- **Headers**: Без авторизации

### 2.5 Фильтрация по состоянию
- **Method**: GET
- **URL**: `{{baseUrl}}/api/listings/condition/New`
- **Headers**: Без авторизации
- **Возможные значения condition**:
  - `New` - Новые
  - `Used` - Б/у

### 2.6 Фильтрация по диапазону цен
- **Method**: GET
- **URL**: `{{baseUrl}}/api/listings/price-range?minPrice=50000&maxPrice=100000`
- **Headers**: Без авторизации
- **Параметры**:
  - `minPrice` - минимальная цена
  - `maxPrice` - максимальная цена

### 2.7 Расширенная фильтрация
- **Method**: POST
- **URL**: `{{baseUrl}}/api/listings/filter`
- **Headers**:
  ```
  Content-Type: application/json
  ```
- **Body** (raw JSON):
```json
{
    "brand": "Apple",
    "condition": "New",
    "minPrice": 50000,
    "maxPrice": 100000
}
```

### 2.8 Получение объявлений пользователя
- **Method**: GET
- **URL**: `{{baseUrl}}/api/listings/user`
- **Headers**:
  ```
  Authorization: Bearer {{authToken}}
  ```

### 2.9 Обновление объявления
- **Method**: PUT
- **URL**: `{{baseUrl}}/api/listings/{{listingId}}`
- **Headers**:
  ```
  Content-Type: application/json
  Authorization: Bearer {{authToken}}
  ```
- **Body** (raw JSON):
```json
{
    "title": "iPhone 14 Pro Max",
    "brand": "Apple",
    "price": 99990.00,
    "condition": "New"
}
```

### 2.10 Удаление объявления
- **Method**: DELETE
- **URL**: `{{baseUrl}}/api/listings/{{listingId}}`
- **Headers**:
  ```
  Authorization: Bearer {{authToken}}
  ```

## Этап 4: Тестирование ошибок

### 4.1 Ошибки при создании объявления

**Попытка создать без токена**:
- **Ожидаемый ответ**: 401 Unauthorized

**Попытка создать с неверным condition**:
```json
{
    "title": "Test",
    "brand": "Test",
    "price": 1000,
    "condition": "Invalid"
}
```
- **Ожидаемый ответ**: 400 Bad Request

**Попытка создать с отрицательной ценой**:
```json
{
    "title": "Test",
    "brand": "Test",
    "price": -1000,
    "condition": "New"
}
```
- **Ожидаемый ответ**: 400 Bad Request

### 4.2 Ошибки при обновлении/удалении

**Попытка обновить чужое объявление**:
- Создайте объявление под одним пользователем
- Попробуйте обновить его под другим пользователем
- **Ожидаемый ответ**: 403 Forbidden

**Попытка обновить несуществующее объявление**:
```
PUT {{baseUrl}}/api/listings/999999
```
- **Ожидаемый ответ**: 404 Not Found

## Полезные советы

1. **Используйте переменные Postman** для автоматического сохранения токенов и ID
2. **Создайте несколько пользователей** для тестирования защиты прав доступа
3. **Тестируйте все комбинации фильтров** для проверки корректной работы
4. **Проверяйте пагинацию** с большим количеством созданных объявлений
5. **Используйте Tests в Postman** для автоматической валидации ответов

## Примеры Tests для Postman

**Проверка успешного создания объявления**:
```javascript
pm.test("Status code is 200", function () {
    pm.response.to.have.status(200);
});

pm.test("Response has id", function () {
    const responseJson = pm.response.json();
    pm.expect(responseJson).to.have.property('id');
    pm.expect(responseJson.id).to.be.a('number');
});

pm.test("Response has correct fields", function () {
    const responseJson = pm.response.json();
    pm.expect(responseJson).to.have.all.keys(
        'id', 'title', 'brand', 'price', 'condition', 'status', 'ownerId', 'createdAt', 'updatedAt'
    );
});
```

**Проверка фильтрации**:
```javascript
pm.test("Filter by brand returns correct results", function () {
    const responseJson = pm.response.json();
    responseJson.content.forEach(function(item) {
        pm.expect(item.brand).to.equal("Apple");
    });
});
